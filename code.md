## Design Pattern Recognition

**Critical:** These designs show visual intent only. You must recognize UI patterns and implement with the correct React Native / Expo Router primitives.

### Navigation Pattern Recognition

Analyze the design to identify navigation structure:

| If the design shows... | Implement as... |
|------------------------|-----------------|
| Bar at bottom with 3-5 icons/labels | `expo-router Tabs` layout |
| Top bar with back arrow and title | `expo-router Stack` with header |
| Hamburger menu or side panel | `expo-router Drawer` layout |
| Horizontal tabs at top of content area | `react-native-tab-view` or custom component |
| Card/sheet sliding from bottom | Modal route or `@gorhom/bottom-sheet` |
| Fullscreen overlay | Modal route with `presentation: 'modal'` |
| Floating circular button (FAB) | Absolute positioned `Pressable`, not navigation |

### Inferring Route Structure

When multiple screens are provided:
1. **Identify shared navigation** — screens with same bottom bar = Tab siblings
2. **Identify parent-child** — screen with back button = Stack child
3. **Identify modals** — overlays, sheets, dialogs = modal routes
4. **Identify drawer items** — hamburger menu destinations = Drawer screens

Example structure inference:
- Home, Search, Profile (all show bottom bar) → Tabs siblings
- ProductDetail (shows back arrow) → Stack child of Home tab
- Settings (accessed from menu) → Stack or Drawer screen

### Component Selection

| For this UI pattern... | Use... | Not... |
|------------------------|--------|--------|
| List of similar items (feed, products) | `FlatList` with `keyExtractor` | `ScrollView` + `.map()` |
| Mixed scrollable content | `ScrollView` | `FlatList` for non-list |
| Screen root wrapper | `SafeAreaView` | Plain `View` |
| Loading state | `ActivityIndicator` | Custom unless design specifies |
| Pull-to-refresh indicator | `RefreshControl` prop | Custom gesture |
| Form with keyboard | `KeyboardAvoidingView` wrapper | Ignoring keyboard |
| Swipeable list rows | `Swipeable` from gesture-handler | Custom pan responders |
| Horizontal snap carousel | `FlatList` with `pagingEnabled` | `ScrollView` + manual |

## React → React Native Conversion

| Design (Web) | Implementation (React Native) |
|--------------|-------------------------------|
| `<div>` | `<View>` |
| `<span>`, `<p>` | `<Text>` (required for all text) |
| `<img>` | `<Image source={{uri}}` or require() |
| `<button>` | `<Pressable>` or `<TouchableOpacity>` |
| `<input>` | `<TextInput>` |
| `<a>` | `<Link>` from expo-router |
| Tailwind classes | StyleSheet.create() objects |
| `gap` | `gap` prop (RN 0.71+) |
| hover states | Press states via Pressable |
| CSS gradients | expo-linear-gradient |

**Key differences:**
- Flexbox defaults to `flexDirection: 'column'` (not row)
- All text must be wrapped in `<Text>`
- No CSS units—use numbers for spacing
- Shadows differ per platform (shadowX for iOS, elevation for Android)

## Feature Implementation

**Important:** The designs contain placeholder/hardcoded data. You must:

1. **Identify feature intent** — What functionality does this UI represent?
2. **Implement dynamic data** — Replace static values with state, props, or API data
3. **Add real interactivity** — Buttons trigger actions, forms validate and submit, lists fetch data
4. **Connect the flow** — Implement navigation and data passing between screens

Examples:
- Static list of items → `FlatList` with data fetching/state
- Hardcoded user name → Dynamic from auth context or API
- Form with prefilled values → Controlled inputs with validation
- Number display (e.g., "5") → Actual computed/fetched value

## Navigation

Use **expo-router** for all navigation.

**If the design includes a bottom navigation bar:**
1. Implement as expo-router `Tabs` layout
2. Match icons, labels, and active states from design
3. Use `screenOptions` for basic tab bar styling:

```tsx
<Tabs screenOptions={{
  tabBarActiveTintColor: '#...',
  tabBarInactiveTintColor: '#...',
  tabBarStyle: { ... }
}}>
```

4. For complex tab bar designs (custom shapes, floating bar, special indicators), use the `tabBar` prop with a custom component:

```tsx
<Tabs tabBar={(props) => <CustomTabBar {...props} />}>
```
