GoodFor Feature Brief

🛒 Smart Basket & Shopping Streak System

Objective

Create a Smart Basket feature that allows users to build a shopping basket while scanning products, manually adding previous items, and improving their Basket Health Score before finalising the shop.

The system should:

• Encourage more product scans
• Encourage healthier swaps
• Encourage weekly usage
• Reward improvement over time
 
1. Basket Flow Overview

Step 1 — Scan Product

User scans a product.

On the analysis screen:

Button:

Add to Basket

 
Step 2 — Basket Builds in Real Time

Every product added updates:

• Basket Health Score
• Colour distribution
• Basket visual

Example:

Basket Score
78 / 100

Colour distribution:

🟢 60% Excellent/Good
🟡 25% Moderate
🔴 15% Improve

 
Step 3 — Basket Can Be Edited

Before checkout, users can:

• Add more products
• Remove products
• Replace products
• Add items from previous baskets
• Add items from favourites

This prevents forcing users to re-scan.

 
2. Adding Products Without Scanning

Users should be able to add items from:

A. Favourites

Button:

Add to Basket

 
B. Previous Shops

Example:

Previous Basket – 8 June
Add product

Tap → item added to current basket.

 
C. Product History

Recent scans list.

 
This reduces friction and increases basket completion.

5. Finalising a Basket

User taps:

Finish Shopping

This locks the basket for that day.

Example message:

Basket Finalised
Score: 82
Nice improvement from your last shop.

The basket is saved with:

• date
• score
• product list

Users can view previous shops.

Example:

5 June — Score 74

Tap to open each basket.

 
7. Improvement Tracking

Show users how their shopping is improving.

Example:

“Great improvement from your last shop.”

 
8. Shopping Streaks

This is the main gamification layer.

Users earn streaks when they log baskets regularly.

Example:

Healthy Shopping Streak
3 weeks

Conditions:

User finalises at least one basket per week.

 
9. Improvement Streaks

Reward users for better shopping choices.

Example:

Score Improvement

Lumi message:

“You improved your basket score by 5 points.”

Inside the basket screen show short tips:

Example:

12. Remove / Replace Logic

Each item allows:

Remove item
Replace with another scan

Score updates instantly.

 
13. Empty Basket State

If user has not added anything yet:

Start building your basket

Scan products or add items from favourites
to see your basket health score.

Button:

Start Scanning

 
14. Daily Basket Logic

One basket per day by default.

User can:

• Start new basket
• Continue current basket

Example:

Continue today's basket

 
15. Key Behaviour Loop

The feature should drive this behaviour:

1. Scan product
2. Add to basket
3. Basket score updates
4. User scans more items
5. User swaps lower scoring items
6. Basket score improves
7. User finalises basket
8. User earns streak
This builds habitual use during shopping.

 
16. Tone & Design Style

The system should feel:

• supportive
• encouraging
• informative

Avoid language like:

❌ bad
❌ unhealthy

Use:

✔ improve
✔ better option
✔ good choice

 
17. Success Metric

The feature should increase:

• scans per session
• basket completions
• weekly active users

Goal:

Users should open GoodFor every time they shop.
